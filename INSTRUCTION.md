# Инструкция по запуску RINEX Downloader на Windows

---

## Шаг 1 — Установка Python

1. Открой браузер и перейди на сайт: **https://python.org/downloads**
2. Нажми большую кнопку **«Download Python 3.x.x»**
3. Запусти скачанный установщик
4. ⚠️ **Обязательно** поставь галочку **«Add Python to PATH»** внизу окна установщика
5. Нажми **«Install Now»** и дождись завершения

---

## Шаг 2 — Установка библиотек

Открой командную строку: нажми **Win + R**, введи `cmd`, нажми Enter.

В открывшемся окне введи команду и нажми Enter:

```
pip install requests beautifulsoup4 lxml
```

Дождись сообщения `Successfully installed ...` — это значит всё установлено.

Библиотеки которые устанавливаются:
- **requests** — отправка запросов к сайту
- **beautifulsoup4** — парсинг HTML страниц
- **lxml** — парсер для beautifulsoup

---

## Шаг 3 — Настройка конфига

Открой файл `rinex_parser.py` в любом текстовом редакторе (например Блокнот или VS Code).

В самом начале файла найди раздел **«КОНФИГУРАЦИЯ»**:

```python
LOGIN    = "YourLogin"                # ← твой логин на bp.eft-cors.ru
PASSWORD = "YourPassword"             # ← твой пароль

OUTPUT_DIR  = Path("rinex_files") # ← папка куда сохраняются файлы
MAX_FILES   = 10                  # ← лимит сайта, не меняй
REQ_DELAY   = 5                   # ← пауза между запросами (сек)
```

**Что можно менять:**

| Параметр | Что делает | Пример |
|---|---|---|
| `LOGIN` | Твой email на сайте | `"my@mail.ru"` |
| `PASSWORD` | Твой пароль | `"mypassword"` |
| `OUTPUT_DIR` | Папка для файлов | `Path("C:/rinex")` |
| `REQ_DELAY` | Пауза между запросами | `5` (меньше не надо) |

**Временные диапазоны** — найди список `ALL_TIME_RANGES`:

```python
ALL_TIME_RANGES = [
    {"start": "02/02/2026 12:00", "end": "02/02/2026 14:00", "tag": "12-14"},
    {"start": "02/02/2026 12:00", "end": "02/02/2026 16:00", "tag": "12-16"},
    {"start": "02/02/2026 12:00", "end": "02/02/2026 18:00", "tag": "12-18"},
]
```

Можно добавлять свои строки по аналогии. Формат даты: `ДД/ММ/ГГГГ ЧЧ:ММ`.  
`tag` — это название подпапки куда сохранятся файлы с этим диапазоном.

**Список станций** — найди `ALL_STATIONS`. Там уже прописаны все 19 работающих станции МСК и МО. Менять не нужно — выбор происходит интерактивно при запуске. Но при желании можно составить свой список станций, добавляя строчку по формату описанному ниже:

```python
{"code": "DMTR", "name": "Дмитров"},
```

## Шаг 4 — Запуск программы

1. Положи файл `rinex_parser.py` в любую удобную папку, например `C:\rinex_parser\`
2. Открой командную строку (`Win + R` → `cmd`)
3. Перейди в папку со скриптом командой `cd`:

```
cd C:\rinex_parser
```

4. Запусти скрипт:

```
python rinex_parser_v3.py
```

## Шаг 5 — Работа с программой

После запуска программа сначала авторизуется, затем задаст два вопроса.

### Вопрос 1 — Выбор временных диапазонов

```
══════════════════════════════════════════════════════════════
  Доступные временные диапазоны:
  ────────────────────────────────────────
  1.  02/02/2026 12:00  →  02/02/2026 14:00
  2.  02/02/2026 12:00  →  02/02/2026 16:00
  3.  02/02/2026 12:00  →  02/02/2026 18:00
  ────────────────────────────────────────
  Enter без ввода → все диапазоны
══════════════════════════════════════════════════════════════
  Выберите диапазоны (напр. «1 2» или «1-3»):
```

в конфиге также можно настроить свои диапазоны (см. Шаг 3)

Варианты ввода:
- `1 2` — только диапазоны 1 и 2
- `1-3` — все три диапазона
- просто **Enter** — выбрать все

### Вопрос 2 — Выбор станций

```
══════════════════════════════════════════════════════════════
  Доступные станции МСК/МО:
  №     Код      Название
  ────────────────────────────────────────
  1     VOSK     Воскресенск
  2     RUZA     Руза
  3     SPSS     Сергиев Посад
  ...
  19    MZSK     Можайск
  ────────────────────────────────────────
  Лимит: 10 файлов/час (при превышении — автокулдаун 62 мин)
  Enter без ввода → все станции (19 шт.)
══════════════════════════════════════════════════════════════
  Выберите станции (напр. «1 2 3» или «1-5»):
```

Варианты ввода:
- `1 3 5` — станции №1, №3, №5
- `1-10` — станции с 1 по 10
- `1,3,7-9` — комбинация
- просто **Enter** — все станции

**Если выбрано больше файлов чем лимит** — программа покажет расчёт и спросит подтверждение:

```
  ⚠  20 станций × 3 диапазона = 60 файлов
     Будет 6 волн с кулдауном 62 мин. Общее время: ~372 мин (~6ч 12мин)
  Продолжить? [y/n]:
```

Нажми `y` и Enter — программа запустится и будет работать автономно.

## Шаг 6 — Автоматический кулдаун

Сайт разрешает скачивать **не более 10 файлов в час**. Программа это учитывает автоматически:

- После 10 скачанных файлов запускается **таймер обратного отсчёта** на 62 минуты
- Таймер обновляется каждую секунду прямо в терминале:

```
  ⏱  Осталось до продолжения: 58:32
```

- После кулдауна программа **сама продолжает** с того места где остановилась
- **Не нужно ничего делать** — просто оставь окно открытым

Если лимит был исчерпан **до запуска** программы — она определит это автоматически (первые 3 попытки упадут) и тоже уйдёт на кулдаун.

Остановить программу в любой момент можно нажатием **Ctrl + C** — уже скачанные файлы сохранятся.

## Где лежат скачанные файлы

Файлы сохраняются в папку `rinex_files` рядом со скриптом, **разложенные по подпапкам** в зависимости от временного диапазона:

```
📁 C:\rinex_parser\
│
├── rinex_parser.py
├── rinex_parser.log       ← лог всех действий программы
│
└── 📁 rinex_files\
      ├── 📁 12-14\        ← файлы с измерением 12:00–14:00
      │     ├── VOSK_020226.0900_020226.1059_xxxxx.26o
      │     ├── DMTR_020226.0900_020226.1059_xxxxx.26o
      │     └── ...
      │
      ├── 📁 12-16\        ← файлы с измерением 12:00–16:00
      │     ├── VOSK_020226.0900_020226.1259_xxxxx.26o
      │     └── ...
      │
      └── 📁 12-18\        ← файлы с измерением 12:00–18:00
            ├── VOSK_020226.0900_020226.1459_xxxxx.26o
            └── ...
```

Имя файла содержит: **код станции** + **дата и время начала** + **дата и время конца** + случайный номер.

Время в имени файла и в заголовке RINEX указано в **UTC** (московское время минус 3 часа) — это стандарт формата RINEX 2.11.

## Частые проблемы

| Проблема | Решение |
|---|---|
| `python не является внутренней командой` | Переустанови Python с галочкой «Add to PATH» |
| `No module named 'requests'` | Выполни `pip install requests beautifulsoup4 lxml` |
| `✗ Авторизация не прошла` | Проверь логин и пароль в конфиге |
| Программа застряла на «Ожидаем SSE» | Станция временно недоступна, попробуй другую |
| Файл скачался 0 байт | Сайт вернул ошибку, детали в `rinex_parser.log` |
